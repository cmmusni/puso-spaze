## 🔌 BACKEND DEPLOYMENT (Railway)

### Create Railway Project:
1. [ ] Go to https://railway.app
2. [ ] Sign up with GitHub
3. [ ] Click "Create Project" → "Deploy from GitHub"
4. [ ] Select `puso-spaze` repository
5. [ ] Set root directory to `/server`

### Configure Environment Variables in Railway:

**From Railway Dashboard → Your Project → Variables:**

```
DATABASE_URL=postgresql://[auto-generated by Railway]
NODE_ENV=production
PORT=3000
OPENAI_API_KEY=[your OpenAI key from platform.openai.com]
GMAIL_USER=[your Gmail address]
GMAIL_PASS=[Gmail app-specific password - NOT regular password]
CORS_ORIGIN=https://puso-spaze.org
```

**How to get Gmail app-specific password:**
1. Go to: https://myaccount.google.com/security
2. Enable 2-factor authentication (if not enabled)
3. Go to "App passwords" 
4. Select Mail + Windows Computer
5. Copy the generated 16-character password
6. Use that as `GMAIL_PASS`

### Add Custom Domain:
- [ ] In Railway → Settings → Domains
- [ ] Add domain: `api.puso-spaze.org`
- [ ] Copy CNAME record provided by Railway

### Deploy:
- [ ] Click deploy
- [ ] Check build logs (should see "✓ Build complete")
- [ ] Verify: `curl https://api.puso-spaze.org/api/posts`

## 🔗 DNS CONFIGURATION (Squarespace)

### Access DNS Settings:
1. [ ] Go to: https://www.squarespace.com
2. [ ] Login → Select `puso-spaze.org`
3. [ ] Click "Settings" → "Domains"
4. [ ] Click on `puso-spaze.org` → "Manage DNS"

### Add DNS Records:

**Option A: Using Vercel Nameservers (Recommended)**
- [ ] In Vercel → Settings → Domains
- [ ] Click `puso-spaze.org`
- [ ] Vercel will show 4 nameservers
- [ ] In Squarespace DNS settings, update nameservers to Vercel's

**Option B: Manual CNAME Records**

From Vercel dashboard, add:
- [ ] `CNAME` record: `www` → `cname.vercel-dns.com`
- [ ] `A` record: `@` → `76.76.19.61` (example; Vercel will specify)

From Railway dashboard, add:
- [ ] `CNAME` record: `api` → `[Railway-provided-URL]`

## 🛠️ TROUBLESHOOTING

### Domain not working (waiting for DNS):
- DNS changes take 24-48 hours
- Check propagation: https://dnschecker.org
- Use `nslookup puso-spaze.org` in terminal

### Frontend can't reach backend:
- Verify `REACT_APP_API_URL` in Vercel env vars
- Check `CORS_ORIGIN` in Railway env vars matches domain
- View browser console for CORS errors

### Build fails:
- Check logs in Railway/Vercel dashboards
- Verify all dependencies installed: `npm install`
- Test build locally: `npm run build`

### Database migration fails:
- In Railway console, run: `npx prisma migrate deploy`
- Check database connection: `DATABASE_URL` format correct
