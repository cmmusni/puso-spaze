📋 Overview

PUSO Spaze (from Filipino "Puso" meaning "Heart") is a compassionate mental health support platform that provides a safe, anonymous space for individuals to share their struggles, receive encouragement, and find hope through community support and AI-powered biblical encouragement.

✅ Live Now: Mobile app available via Expo Go and Production builds. Backend API deployed on Railway.

🎯 What Can You Do?

  • Share Anonymously: Post your thoughts and struggles without revealing your identity
  • Receive Hourly Hope: Get AI-generated biblical encouragement posts every hour from our 🕊️ Hourly Hope system
  • Engage Safely: React with emojis and comment on posts to support others
  • Stay Notified: Receive push notifications for reactions, comments, and new encouragement posts
  • Manage Your Identity: Device-bound accounts ensure privacy while allowing username customization
  • Coach Access: Special dashboard for mental health coaches to send invites and monitor community health

Database Schema

The application uses PostgreSQL with the following key models:

  • User: User accounts with role, displayName, expoPushToken
  • Post: User posts with content, tags, moderationStatus
  • Comment: Comments on posts
  • Reaction: Emoji reactions (HEART, PRAY, STRONG, STAR, SMILE)
  • Notification: Push notifications (NEW_COMMENT, NEW_REACTION, NEW_ENCOURAGEMENT)

API Endpoints

Route Method Description
/api/auth/register POST Register new user (device binding)
/api/posts GET Get all posts with pagination
/api/posts/:id GET Get single post by ID
/api/posts POST Create new post
/api/posts/:id/comments POST Add comment to post
/api/posts/:id/reactions POST Add reaction to post
/api/notifications GET Get user notifications
/api/notifications/:id/read PATCH Mark notification as read
/api/users/:id PATCH Update username

Key Features Implementation

🕊️ Hourly Hope System

  • Scheduler: node-cron runs every hour ('0 * * * *')
  • AI Generation: OpenAI GPT-4o-mini creates 280-character encouragement posts
  • Fallback System: 8 pre-written verses ensure posts even if API fails
  • System User: Posts created by 'system-encouragement-bot' user with COACH role
  • Notifications: All users notified when new encouragement is posted

🔔 Notification System

  • Push Delivery: Expo Server SDK sends push notifications to devices
  • ES Module Fix: Dynamic import for expo-server-sdk compatibility with CommonJS
  • Notification Types: NEW_COMMENT, NEW_REACTION, NEW_ENCOURAGEMENT
  • Clickable Navigation: Tapping notification opens the related post
  • Unread Tracking: Badge counter shows unread notifications

🔐 Device-Bound Authentication

  • Device Owner: First username on device is permanently bound
  • Secure Storage: SecureStore (native) or AsyncStorage (web)
  • Username Updates: Allowed, automatically updates device owner too
  • Validation: Prevents different usernames from using same device

🔒 Privacy & Data

What Information We Store

Account Information

  • Device Owner: Username bound to device (cannot be shared)
  • Display Name: Customizable username (can be updated)
  • User Role: USER, COACH, or ADMIN
  • Expo Push Token: For sending notifications

Content Data

  • Posts: Your shared thoughts and struggles
  • Comments: Your responses to others' posts
  • Reactions: Emoji reactions you've given
  • Moderation Status: Automated content safety check

Notification Data

  • Notification History: Record of notifications sent to you
  • Read Status: Which notifications you've viewed
  • Related Content: Links to posts/comments that triggered notifications
✅ Your Privacy Matters: We never sell your personal information. All data is encrypted and stored securely. Anonymous posting ensures your identity stays private. Device binding prevents unauthorized access to your account.

For Developers

Local Development Setup

# Clone repository
git clone https://github.com/cmmusni/puso-spaze.git
cd puso-spaze

# Setup backend
cd server
npm install
cp .env.example .env
# Edit .env with your credentials
npx prisma generate
npx prisma migrate dev
npm run dev

# Setup mobile app (new terminal)
cd apps/mobile
npm install
npx expo start

Required Environment Variables

# Backend (.env)
DATABASE_URL=postgresql://user:pass@localhost:5432/puso_spaze
